## Sites Where SOM Is Larger Than HTML

These are minimal/static pages where the SOM structure adds overhead that outweighs the compression:

| Site | SOM overhead | Notes |
|---|---|---|
| producthunt.com | +1137% | JS SPA — curl returns near-empty shell, Plasmate renders more |
| crates.io | +424% | Very minimal static HTML |
| example.com | +118% | 152-token toy page |
| python.org | +62% | Lightweight static homepage |
| postgresql.org | +47% | Minimal HTML |
| jsonplaceholder.typicode.com | +32% | API docs page, minimal |
| news.ycombinator.com | +20% | HN's notoriously minimal HTML |
| pypi.org | +5% | Borderline |

**Pattern**: Plasmate wins on SPAs and JS-heavy pages. It loses on sites that already serve minimal HTML. For sites under ~5K HTML tokens, consider raw fetch instead.

## Category Breakdown

### SaaS / Cloud (excellent — 30-117x)
cloud.google.com, linear.app, figma.com, vercel.com, aws.amazon.com, azure.microsoft.com, stripe.com/docs, tailwindcss.com, notion.so, docker.com — all 30x+ compression.

### Frameworks / Docs (strong — 7-54x)
nodejs.org (39x), react.dev (11x), angular.dev (7x), nextjs.org (22x), typescriptlang.org (24x), kubernetes.io (2.5x), docs.github.com (4x).

### News (variable — 13-110x)
NYT (110x), Guardian (17x), Wired (26x), BBC (13x), Ars Technica (10x). High variance — depends on how much inline JS/CSS is bundled.

### Minimal HTML (SOM loses)
HN, Python.org, pypi.org, crates.io, example.com — already-lean sites where SOM adds overhead.

## Notes

- `run-benchmark.sh` had a JSON serialization bug: `bc` outputs bare decimals (`.4`) which are invalid JSON. Fixed in this run; bug report in PR description.
- Product Hunt returned near-empty HTML via curl (JS SPA) but Plasmate rendered the full page, giving an inflated SOM:HTML ratio in the wrong direction.
- The 7.1M HTML tokens across 36 pages cost ~$21.44 at GPT-4 rates ($3/1M). Plasmate reduces that to ~$1.22 — **82% cost reduction** on these 36 pages at today's model pricing.
